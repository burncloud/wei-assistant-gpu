# 概述
本程序用于记录供应商提供的机器信息，便于客户查询所有详细数据。适用于需要管理多家供应商及其服务器资源的企业或个人，提升信息透明度和管理效率。

# 核心功能
- 供应商及机器信息录入
  - 记录联系人、微信、手机、位置、价格、地区、交付条件等详细字段
  - 支持命令行JSON整体传入和常规参数传入两种方式
- 数据查询
  - 客户可查询所有数据库字段的详细信息
- 数据库自动初始化
  - 首次运行自动创建sqlite数据库及表结构，字段带注释
  - 数据库文件wei-assistant.db存在则跳过初始化
- 测试代码
  - 提供自动化测试，确保主要功能可用

# 用户体验
- 用户画像：
  - 供应商信息录入员
  - 资源查询客户
- 关键流程：
  - 录入员通过命令行添加供应商及机器信息
  - 客户通过命令行查询所有详细信息
- 交互方式：
  - 命令行工具，支持JSON和参数两种输入
  - 查询结果以表格或结构化文本输出

# 技术架构
- 组件：
  - 命令行主程序
  - SQLite数据库（wei-assistant.db）
  - 数据库初始化模块
  - 数据插入与查询模块
  - 测试模块
- 数据模型：
  - 表：suppliers
    - 字段：编号（主键，自增）、联系人、微信、手机号码、数量、位置、价格、带宽价格、存储价格、最低签约周期、违约罚则、付款条件、服务器名称、服务器配置、租赁模式、组网类别（所有字段需带注释）
- API/集成：
  - 命令行参数解析（支持JSON和参数）
- 基础设施：
  - 仅需本地sqlite，无需外部依赖

# 开发路线图
- MVP：
  - 数据库自动初始化
  - 命令行数据录入（JSON/参数）
  - 数据查询输出
  - 基本测试用例
- 后续增强：
  - 支持数据导出（如CSV）
  - 更丰富的查询筛选
  - 多用户权限管理

# 逻辑依赖链
- 先实现数据库结构和初始化
- 再实现命令行参数解析与数据录入
- 最后实现查询与测试

# 风险与应对
- 技术挑战：命令行参数多样性，需详细测试
- MVP定义：优先保证录入与查询主流程
- 资源约束：本地单文件sqlite，易于部署

# 附录
- 字段注释与示例：
  - 编号：主键，自增
  - 联系人：供应商联系人姓名
  - 微信：联系人微信号
  - 手机号码：联系人手机号
  - 数量：可用机器数量
  - 位置：服务器物理位置
  - 价格：整体价格
  - 带宽价格：带宽单价
  - 存储价格：存储单价
  - 最低签约周期：最短租赁周期
  - 违约罚则：提前解约的处罚条款
  - 付款条件：付款方式与周期
  - 服务器名称：设备名称
  - 服务器配置：CPU/内存/硬盘等配置
  - 租赁模式：如包月、包年等
  - 组网类别：如专线、共享等
